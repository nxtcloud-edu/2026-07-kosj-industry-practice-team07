# 고려대 세종 산업체 실습 프로젝트 안내

이 저장소는 2026년 7월 고려대 세종 산업체 실습 프로젝트의 팀별 MVP 개발용 public repository입니다.

## 기본 운영

- 팀별 커뮤니케이션은 Slack 팀 채널을 사용합니다.
- 중요한 논의, 결정사항, 질문, 진행 상황은 Slack 채널 내부 Canvas에 정리합니다.
- 멘토링 전 Canvas를 정리해두면 멘토에게 팀의 맥락을 빠르게 전달할 수 있습니다.
- 발표자료는 운영진이 전달한 팀별 Canva 링크에서 공동 제작합니다.
- MVP 앱의 구현 과정과 결과물은 이 GitHub repository로 확인합니다.

## 주요 일정과 제출물

### 1주차: 초기 아이디어 노트

- 제출 기한: 7/10(금) 14:00까지
- 형식: HTML 페이지
- 내용: RFP 분석, 문제 정의, 핵심 가설, 해결 아이디어, 차별점
- 멘토 피드백: 7/13(월)까지 전달

### 2주차: 입찰 제안서

- 제출 기한: 7/17(금)
- 형식: PDF 파일
- 내용: 문제, 해결방안, 시장/사용자, 수익모델, 실행계획, MVP 범위, 확장 로드맵
- 7/15(수) 온라인 멘토링에서 문제 정의, 앱 구현 준비사항, 제안서 기획 내용을 팀별 Slack 허들로 점검합니다.

### 3주차: MVP 앱 구현

- 이 repository에서 MVP 앱을 구현하고 관리합니다.
- 7/22~24 기간에는 팀이 필요할 경우 멘토링을 신청해 진행합니다.
- README에 프로젝트 소개, 실행 방법, 주요 기능, 팀원 역할을 정리하는 것을 권장합니다.

### 4주차: 발표자료 및 발표 영상

- 전달받은 Canva 링크로 최종 발표자료를 제작합니다.
- 최종 발표자료를 활용해 10~15분 분량의 입찰 제안서 발표 영상을 제작해 제출합니다.

## RFP / REQ 용어

- RFP(Request for Proposal): 발주기관이 원하는 문제, 요구사항, 제약조건을 정리한 제안요청서입니다.
- REQ(Requirement): RFP 안의 세부 요구사항입니다. REQ-01, REQ-02처럼 번호가 붙습니다.
